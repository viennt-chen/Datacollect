# 停机状态匹配规则排查指南

## 问题描述
配置了停机状态匹配规则，topic实际数据是 `353`，但匹配失败。

## 排查步骤

### 1️⃣ 查看后端日志（最重要）

重启后端服务后，访问设备状态监控页面，查看后端控制台输出：

```
[规则评估] 配置#X (stop):
  - Topic: xxx
  - 字段路径: xxx
  - 实际值: 353 (类型: str/int)  ← 关键！
  - 匹配规则: equals
  - 匹配值: 353 (原始)
    [匹配详情] 规则=equals, 实际值=353(str), 匹配值=353(int)
    ⚠️ 值不相等: '353' != 353
  - 匹配结果: ❌ 失败
```

### 2️⃣ 常见问题

#### 问题A：类型不匹配（最常见）
- **现象**: 实际值是字符串 `"353"`，但匹配值配置为数字 `353`
- **解决**: 
  - 方案1：将匹配值改为字符串 `"353"`（加引号）
  - 方案2：使用 `in_range` 规则：`[353, 353]`

#### 问题B：字段路径错误
- **现象**: 实际值为 `None`
- **原因**: 字段路径配置错误，无法从MQTT消息中获取值
- **解决**: 检查topic的JSON结构，确认字段路径正确

#### 问题C：Topic名称错误
- **现象**: 未找到MQTT消息
- **原因**: Topic名称配置错误或设备未关联该Topic
- **解决**: 在设备管理中确认设备已关联正确的Topic

#### 问题D：互斥状态优先级
- **现象**: 规则匹配成功，但状态显示不是停机
- **原因**: 加工/换模/故障状态优先级更高
- **解决**: 调整优先级数字（数字越小优先级越高）

### 3️⃣ 前端调试

点击停机状态卡片，会弹出详细匹配信息：

```
状态类型：停机状态
当前状态：N

○ 未匹配到规则

规则评估详情 (1条)：
────────────────────────────────────────
1. 停机规则 ○
   未匹配（值不匹配或条件不满足）
```

同时打开浏览器控制台（F12），查看日志：
```
[状态详情] stop: {matched: false, evaluated_configs: [...]}
```

### 4️⃣ 快速修复建议

#### 如果实际值是字符串 "353"：
```
匹配规则：equals
匹配值："353"  ← 注意加引号
```

#### 如果实际值是数字 353：
```
匹配规则：equals
匹配值：353  ← 不加引号
```

#### 通用方案（推荐）：
```
匹配规则：in_range
匹配值：[353, 353]  ← 范围匹配，自动转换类型
```

## 验证方法

1. 修改配置后，刷新页面
2. 查看后端日志，确认匹配结果
3. 查看前端状态指示灯是否变为停机状态
